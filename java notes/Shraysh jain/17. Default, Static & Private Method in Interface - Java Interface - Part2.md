## 1. Default Method (Java 8)

Before Java 8, interfaces could only have abstract methods. Adding a new method forced **all** implementing classes to change.

### The Problem

```java
interface Bird {
    void fly();                       // existing
    int getMinFlyHeight();            // NEW вҖ” every class must implement!
}

class Eagle implements Bird { ... }   // must add getMinFlyHeight()
class Sparrow implements Bird { ... } // must add getMinFlyHeight()
// ... 100+ classes need changes!
```

### The Solution: `default` keyword

```java
interface Bird {
    void fly();
    
    default int getMinFlyHeight() {      // implementation IN the interface
        return 100;                       // all classes get this for free
    }
}

Eagle eagle = new Eagle();
eagle.getMinFlyHeight();  // вң… works вҖ” inherited from interface
```

> **Why Java 8?** The `stream()` method was added to the `Collection` interface. Without `default`, hundreds of implementing classes would need modification.

### Rules

| Rule | Detail |
|---|---|
| Has implementation body | вң… Unlike abstract methods |
| Classes can override it | вң… Optional |
| Can be inherited | вң… Available to all implementing classes |

## Diamond Problem with Default Methods

If a class implements **two interfaces** with the **same default method**, it **must** provide its own implementation.

```java
interface Bird       { default void canBreathe() { ... } }
interface LivingThing { default void canBreathe() { ... } }

class Eagle implements Bird, LivingThing {
    @Override
    public void canBreathe() {    // MANDATORY вҖ” resolve ambiguity
        Bird.super.canBreathe();  // can call specific parent if needed
    }
}
```

## Extending Interfaces with Default Methods

Three options when a child interface extends a parent with a default method:

### Option 1: Do Nothing (Inherit as-is)

```java
interface LivingThing { default void canBreathe() { ... } }
interface Bird extends LivingThing { }  // inherits canBreathe()

class Eagle implements Bird {
    // canBreathe() available via inheritance вң…
}
```

### Option 2: Re-declare as Abstract

```java
interface Bird extends LivingThing {
    void canBreathe();  // now abstract again! classes MUST implement
}
```

### Option 3: Override with New Default

```java
interface Bird extends LivingThing {
    @Override
    default void canBreathe() {
        LivingThing.super.canBreathe();  // call parent's default
        // add more logic here
    }
}
```

## 2. Static Method (Java 8)

Provides implementation in the interface, but **cannot be overridden**.

```java
interface Bird {
    static void canBreathe() {
        System.out.println("Birds breathe");
    }
}

// Access via interface name only
Bird.canBreathe();  // вң…

// Cannot override in implementing class вқҢ
class Eagle implements Bird {
    // @Override void canBreathe() { } вҶ’ ERROR!
    
    void someMethod() {
        Bird.canBreathe();  // use it, don't override it
    }
}
```

| Feature | Default | Static |
|---|---|---|
| Has body | вң… | вң… |
| Can be overridden | вң… | вқҢ |
| Access | Via object | Via interface name |
| Modifier | `public` (implicit) | `public` (implicit) |

> **Why `stream()` is `default` not `static`:** Some classes need to override stream behavior вҖ” `static` would prevent that.

## 3. Private Method (Java 9)

Allows **code reuse** within the interface вҖ” shared logic between multiple `default` methods.

```java
interface Bird {
    void fly();  // abstract

    default void eat()  { commonLogic(); /* + eat-specific code */ }
    default void sleep() { commonLogic(); /* + sleep-specific code */ }

    private void commonLogic() {        // Java 9 вҖ” shared helper
        System.out.println("Common setup code");
    }

    private static void staticHelper() { // Java 9 вҖ” static private
        System.out.println("Static helper");
    }
}
```

### Why Private Methods?

| Problem | Solution |
|---|---|
| Multiple `default` methods share 80% code | Extract to `private` helper |
| Code duplication in interface | `private` method = reusable |
| Not part of public API | Stays internal to interface |

### Access Rules

```
  default method (non-static)
    в”ңв”Җв”Җ can call: private вң…
    в”ңв”Җв”Җ can call: private static вң…
    в””в”Җв”Җ can call: static вң…

  static method
    в”ңв”Җв”Җ can call: private static вң…
    в””в”Җв”Җ can call: private вқҢ  (static can't access non-static)
```

### Rules

| Rule | Detail |
|---|---|
| Must have body | вң… Cannot be abstract |
| Accessible outside interface? | вқҢ Only within the interface |
| Can be `static`? | вң… `private static` also allowed |

## Summary

| Concept | Key Point |
|---|---|
| **Default method** | Implementation in interface, overridable, Java 8 |
| **Why default?** | Add methods to legacy interfaces without breaking implementations |
| **Diamond conflict** | Two interfaces with same default вҶ’ class must override |
| **Static method** | Cannot be overridden, accessed via `InterfaceName.method()` |
| **Private method** | Code reuse within interface, Java 9 |
| **Private static** | Can be called from `static` methods in interface |
