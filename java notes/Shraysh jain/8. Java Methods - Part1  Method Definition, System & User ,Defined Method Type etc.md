## What is a Method?

A **method** is a collection of instructions that performs a specific task. It takes input, processes it, and optionally returns output.

```
       ┌─────────────────────────┐
Input ──▶   Method (Task)        ──▶ Output
       │  • Instruction 1        │
       │  • Instruction 2        │
       │  • return result         │
       └─────────────────────────┘
```

### Advantages

| Advantage | Description |
|---|---|
| **Reusability** | Write once, call from multiple places |
| **Readability** | Method name conveys intent (`getInvoice()`, `sum()`) |

```java
public class Calculation {
    // Reusable method
    public int sum(int a, int b) {
        int total = a + b;
        System.out.println("Sum: " + total);
        return total;
    }

    public int getPriceOfPen() {
        return sum(2, 5);   // reused!
    }

    public int getCombinedAge() {
        return sum(2, 5);   // reused again!
    }
}
```

## Method Declaration Syntax

```java
access_specifier  return_type  methodName(parameter_list) throws Exception {
    // method body
    return value;
}
```

| Component | Example | Purpose |
|---|---|---|
| Access Specifier | `public` | Who can access this method |
| Return Type | `int` | What the method returns |
| Method Name | `sum` | Identifier to call the method |
| Parameters | `int a, int b` | Inputs the method uses |
| Body | `{ ... }` | Instructions to execute |
| `throws` | `throws ArithmeticException` | Declares possible exceptions *(covered separately)* |

## Access Specifiers

Java has **4 access levels**:

```
┌──────────────────────────────────────────────────────────────────────┐
│                      Access Specifier Scope                          │
├──────────────┬───────────┬────────────┬─────────────┬────────────────┤
│              │ Same Class│ Same Pkg   │ Subclass in │ Any Package    │
│              │           │ Diff Class │ Diff Pkg    │ Any Class      │
├──────────────┼───────────┼────────────┼─────────────┼────────────────┤
│ public       │     ✅    │     ✅     │     ✅      │     ✅         │
│ protected    │     ✅    │     ✅     │     ✅      │     ❌         │
│ default      │     ✅    │     ✅     │     ❌      │     ❌         │
│ private      │     ✅    │     ❌     │     ❌      │     ❌         │
└──────────────┴───────────┴────────────┴─────────────┴────────────────┘
```

### What is a Package?

A **package** is a logical grouping of related classes:

```
sales_department/          hr/
├── Invoice.java           ├── JobPortal.java
├── Order.java             └── Application.java
```

### `public` — Accessible everywhere

```java
// Package: sales
public void getInvoice() { ... }

// Package: hr — can still access it ✅
Invoice inv = new Invoice();
inv.getInvoice();
```

### `private` — Same class only

```java
class Invoice {
    private void getInvoice() { ... }

    public void printInvoice() {
        getInvoice();  // ✅ same class
    }
}

// From any other class:
new Invoice().getInvoice();  // ❌ Compilation Error
```

### `protected` — Same package + subclasses in different packages

```java
// Package: sales
class Invoice {
    protected void getInvoice() { ... }
}
class Order {
    void test() { new Invoice().getInvoice(); }  // ✅ same package
}

// Package: hr
class JobPortal extends Invoice {   // subclass
    void test() { getInvoice(); }   // ✅ subclass in diff package
}
class App {
    void test() { new Invoice().getInvoice(); }  // ❌ not subclass
}
```

### `default` (no modifier) — Same package only

```java
class Invoice {
    void getInvoice() { ... }  // default: package-private
}
// Same package → ✅ accessible
// Different package → ❌ even if subclass
```

## Return Type

What the method returns after computation.

```java
public int sum(int a, int b) { return a + b; }   // returns int
public boolean isValid() { return true; }          // returns boolean
public void printData() { System.out.println(); }  // returns nothing
```

> For `void` methods, `return;` (without value) is optional — just exits the method.

## Method Name & Naming Convention

| Rule | Example |
|---|---|
| Should be a **verb** (action) | `getInvoice`, `calculateSum`, `printData` |
| Start with **lowercase** | `getEmployeeDetails` (not `GetEmployeeDetails`) |
| Use **camelCase** for multi-word | `getEmployeeDetails` |

## Parameters / Arguments

Variables passed into a method as input. Can be empty.

```java
public int sum(int a, int b, String label) { ... }  // 3 params
public void getInvoice() { ... }                      // 0 params
```

## Types of Methods

### 1. System-Defined Methods

Pre-built methods from Java libraries (provided by JDK/JRE):

```java
double result = Math.sqrt(25);        // java.lang.Math
String upper = "hello".toUpperCase(); // java.lang.String
```

### 2. User-Defined Methods

Created by the programmer:

```java
public int sum(int a, int b) {
    return a + b;
}
```

## Method Overloading (Static Binding)

> **Same name, different parameters** — within the **same class**. Resolved at **compile time**.

```java
class Invoice {
    void getInvoice() { ... }                  // 0 params
    void getInvoice(String type) { ... }       // 1 param (String)
    void getInvoice(int a, int b) { ... }      // 2 params (int, int)
}
```

### Rules

| Rule | Detail |
|---|---|
| Method name | Must be **same** |
| Parameters | Must be **different** (count or type) |
| Return type | **NOT considered** — cannot differentiate by return type alone |

```java
// ❌ INVALID — same name, same params, only return type differs
int getInvoice(String s) { ... }
void getInvoice(String s) { ... }   // Compilation Error!
```

## Method Overriding (Dynamic Binding)

> **Same method in child class** as parent class. Resolved at **runtime**.

```java
class Person {
    public void profession() {
        System.out.println("I am a Person");
    }
}

class Doctor extends Person {
    @Override
    public void profession() {
        System.out.println("I am a Doctor");
    }
}
```

> Everything must be identical: name, return type, access specifier, parameters.

---

## How to Invoke Overloaded & Overridden Methods

### Overloaded — Differentiated by Arguments

```java
Invoice inv = new Invoice();
inv.getInvoice("type1");    // → calls String version
inv.getInvoice(4, 8);       // → calls (int, int) version
```

### Overridden — Resolved at Runtime

```java
Person obj = new Doctor();   // parent reference, child object
obj.profession();            // Output: "I am a Doctor"
```

```
  Compile time: obj type is Person
  Runtime:      obj holds Doctor instance
                → JVM checks Doctor class first
                → profession() found → calls it
                → If not found → falls back to Person
```

## Summary

| Concept | Key Point |
|---|---|
| **Method** | Collection of instructions performing a task |
| **Access Specifiers** | `public` > `protected` > `default` > `private` |
| **Return Type** | What method returns; `void` = nothing |
| **System-Defined** | `Math.sqrt()`, `String.toUpperCase()`, etc. |
| **User-Defined** | Created by programmer |
| **Overloading** | Same name, different params, same class (compile time) |
| **Overriding** | Same method in child class (runtime) |
| **Static Binding** | Overloading — resolved at compile time |
| **Dynamic Binding** | Overriding — resolved at runtime |
