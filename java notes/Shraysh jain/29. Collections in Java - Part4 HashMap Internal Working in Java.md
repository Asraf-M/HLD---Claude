## Why Map is Not a Child of Collection

`Collection` stores **single values** (one element per slot), while `Map` stores **key-value pairs**. The data model is fundamentally different.

```
  Collection Interface          Map Interface
  ┌─────────┐                   ┌──────────────────┐
  │  [10]   │                   │  key1 → value1   │
  │  [20]   │                   │  key2 → value2   │
  │  [30]   │                   │  key3 → value3   │
  └─────────┘                   └──────────────────┘
```

---

## Map Interface Properties

| Property | Detail |
|---|---|
| **Keys** | Must be unique — no duplicates |
| **Values** | Can be duplicate |
| **Duplicate key** | Overwrites the existing value (not error) |
| **Null key** | ✅ One null key allowed (HashMap) |
| **Null values** | ✅ Multiple null values allowed |

## Internal Structure — Node / Entry

HashMap internally uses an **array of Nodes** (also called Entry or bucket array).

```java
static class Node<K, V> {
    int hash;           // hash of the key
    K key;              // the key
    V value;            // the value
    Node<K, V> next;    // pointer to next node (for collision chaining)
}
```

```
  Bucket Array (default size = 16)
  ┌───────────────────────────────────────────┐
  │ Index │  Node                              │
  ├───────┼────────────────────────────────────┤
  │   0   │  null                              │
  │   1   │  [hash|"key1"|"val1"|next] → null  │
  │   2   │  null                              │
  │   3   │  [hash|"key3"|"val3"|next] → [hash|"key7"|"val7"|null]  ← collision!
  │   4   │  null                              │
  │  ...  │  ...                               │
  │  15   │  null                              │
  └───────────────────────────────────────────┘
```

## How put() Works

```
  map.put(key, value)
       │
       ▼
  1. Compute hash = hash(key)
       │
       ▼
  2. Compute index = hash % table.length
       │
       ▼
  3. Go to bucket[index]
       │
       ├── Bucket is EMPTY?
       │       → Create new Node, store here ✅
       │
       └── Bucket is NOT empty? (collision)
               │
               ▼
           Iterate linked list at this bucket
               │
               ├── Found node with SAME key (hash match + equals)?
               │       → Overwrite value ✅
               │
               └── No matching key found?
                       → Append new Node to end of list ✅
```

```java
HashMap<String, Integer> map = new HashMap<>();
map.put("Apple", 1);    // hash("Apple") % 16 = 5 → bucket[5]
map.put("Banana", 2);   // hash("Banana") % 16 = 9 → bucket[9]
map.put("Apple", 10);   // hash("Apple") % 16 = 5 → found "Apple" → overwrite to 10
```

## How get() Works

```
  map.get(key)
       │
       ▼
  1. Compute hash = hash(key)
       │
       ▼
  2. Compute index = hash % table.length
       │
       ▼
  3. Go to bucket[index]
       │
       ├── Bucket is EMPTY?
       │       → return null
       │
       └── Bucket has nodes?
               │
               ▼
           Iterate linked list
               │
               ├── hash matches AND key.equals(nodeKey)?
               │       → return node.value ✅
               │
               └── End of list, no match?
                       → return null
```

## hashCode & equals Contract

Both `hashCode()` and `equals()` are used together to locate and identify keys.

```
  Step 1: hashCode() → find the BUCKET (index)
  Step 2: equals()   → find the exact NODE in that bucket
```

### The Contract

| Rule | Detail |
|---|---|
| If `a.equals(b)` is true | `a.hashCode() == b.hashCode()` **must** be true |
| If `a.hashCode() == b.hashCode()` | `a.equals(b)` is **NOT necessarily** true (collision) |
| If `a.equals(b)` is false | hashCodes may or may not match |

```
  Same hashCode + equals = true  → SAME key (overwrite value)
  Same hashCode + equals = false → DIFFERENT key (collision → chain)
  Different hashCode              → DIFFERENT bucket (no collision)
```

> **Always override both** `hashCode()` and `equals()` when using custom objects as HashMap keys.

## Load Factor & Rehashing

### Load Factor

The **threshold ratio** at which the HashMap resizes.

```
  Default capacity:    16
  Default load factor: 0.75
  Threshold:           capacity × load factor = 16 × 0.75 = 12
```

When the number of entries exceeds the threshold → **rehashing** occurs.

### Rehashing Process

```
  Before: capacity = 16, entries = 12 (threshold reached!)
       │
       ▼
  1. Double the capacity: 16 → 32
  2. Recalculate index for ALL existing entries
     (because index = hash % capacity, and capacity changed)
  3. Redistribute entries into new bucket array
       │
       ▼
  After: capacity = 32, new threshold = 32 × 0.75 = 24
```

| Property | Value |
|---|---|
| Default capacity | 16 |
| Default load factor | 0.75 |
| Resize trigger | entries > capacity × loadFactor |
| New capacity | 2× old capacity |
| Cost | O(n) — must rehash and move all entries |

> **Lower load factor** = more memory, fewer collisions, faster lookups.
> **Higher load factor** = less memory, more collisions, slower lookups.

## Treeification (Java 8)

When a single bucket's **linked list** exceeds **8 nodes**, it converts to a **Red-Black Tree** for better worst-case performance.

```
  Linked List (≤ 8 nodes)          Red-Black Tree (> 8 nodes)
  ┌──┐ → ┌──┐ → ┌──┐ → ...        ┌──────┐
  │N1│   │N2│   │N3│               │  N4  │
  └──┘   └──┘   └──┘              ╱       ╲
                                ┌──┐     ┌──┐
  Lookup: O(n)                  │N2│     │N6│
                                └──┘     └──┘
                               Lookup: O(log n)
```

| Feature | Linked List | Red-Black Tree |
|---|---|---|
| Structure | Linear chain | Balanced BST |
| Threshold | ≤ 8 nodes | > 8 nodes (treeify) |
| Lookup time | O(n) | O(log n) |
| Untreeify | — | < 6 nodes → back to linked list |

### Time Complexity Summary

| Operation | Average | Worst Case |
|---|---|---|
| `put()` | O(1) | O(log n) |
| `get()` | O(1) | O(log n) |
| `remove()` | O(1) | O(log n) |
| `containsKey()` | O(1) | O(log n) |

## Common Methods

```java
HashMap<String, Integer> map = new HashMap<>();

// Insert
map.put("Apple", 1);                   // add or overwrite
map.putIfAbsent("Apple", 99);          // insert ONLY if key absent (won't overwrite)

// Retrieve
map.get("Apple");                      // 1 (or null if absent)
map.getOrDefault("Mango", 0);         // 0 (key not found, return default)
map.containsKey("Apple");             // true

// Remove
map.remove("Apple");                   // removes entry, returns value

// Size
map.size();                            // number of entries
map.isEmpty();                         // true if empty

// Iteration
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    entry.getKey();                    // key
    entry.getValue();                  // value
}

for (String key : map.keySet()) { }    // iterate keys only
for (Integer val : map.values()) { }   // iterate values only
```

## HashMap vs Hashtable

| Feature | HashMap | Hashtable |
|---|---|---|
| **Thread-safe** | ❌ No | ✅ Yes (synchronized) |
| **Null key** | ✅ One allowed | ❌ Not allowed |
| **Null values** | ✅ Allowed | ❌ Not allowed |
| **Performance** | Faster (no sync overhead) | Slower (synchronized) |
| **Modern alternative** | — | `ConcurrentHashMap` (better concurrency) |

```
  HashMap  ─── Not thread-safe ─── Fastest
     │
     ├── Hashtable ─── Thread-safe (synchronized) ─── Legacy
     │
     └── ConcurrentHashMap ─── Thread-safe (segment locking) ─── Preferred
```

## Summary

| Concept | Key Point |
|---|---|
| **Map** | Key-value pairs — not part of Collection hierarchy |
| **Node** | Internal structure: hash + key + value + next |
| **put()** | Hash → index → store (or chain on collision, overwrite on duplicate key) |
| **get()** | Hash → index → iterate → equals check → return value |
| **hashCode + equals** | hashCode finds bucket, equals finds exact node |
| **Load factor** | 0.75 default — rehash at threshold (doubles capacity) |
| **Treeification** | > 8 collisions → linked list → Red-Black Tree → O(log n) |
| **Average O(1)** | Worst case O(log n) after Java 8 treeification |
