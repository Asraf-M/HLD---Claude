## What is an Interface?

An interface helps **two systems interact** without one knowing the internal details of the other — i.e., it achieves **abstraction**.

```
  System 1                    System 2
  ┌──────────┐     Interface     ┌──────────────────┐
  │          │ ──── fly() ─────▶ │  (hidden logic)  │
  │  Caller  │ ──── eat() ─────▶ │  (hidden logic)  │
  └──────────┘                   └──────────────────┘

  System 1 doesn't know HOW fly/eat work — only THAT they exist.
```

**Real-world analogy:** Pressing the brake pedal (interface) — you don't know how the car internally slows down.

## How to Define an Interface

```java
[modifier] interface InterfaceName [extends Interface1, Interface2, ...] {
    // body: abstract methods, constants
}
```

| Component | Rules |
|---|---|
| Modifier | Only `public` or *default* (package-private). No `private` / `protected` |
| `extends` | Can extend **multiple interfaces** (comma-separated) |
| Cannot extend | Classes — only interfaces |

```java
interface Bird { void fly(); }

interface LivingThing { boolean canBreathe(); }

interface NonFlyingBird extends Bird, LivingThing {
    void canRun();
}
```

## Why We Need Interfaces

### 1. Abstraction (100%)

Defines **what** a class must do, but **not how**.

```java
interface Bird {
    void fly();     // signature only — no implementation
}

class Eagle implements Bird {
    @Override
    public void fly() {
        System.out.println("Eagle soars high");  // HOW is here
    }
}
```

### 2. Polymorphism

Interface can be used as a **data type** — resolved at runtime.

```java
Bird b1 = new Eagle();   // holds Eagle
Bird b2 = new Hen();     // holds Hen

b1.fly();  // → Eagle's fly()   (runtime lookup)
b2.fly();  // → Hen's fly()     (runtime lookup)
```

> You **cannot** create an object of an interface (`new Bird()` ❌), but it can **hold references** to implementing classes.

### 3. Multiple Inheritance (via Interfaces only)

Java does NOT support multiple inheritance with classes (diamond problem), but **does** with interfaces.

```
  ┌──────────────┐     ┌──────────────┐
  │ WaterAnimal  │     │ LandAnimal   │
  │ canBreathe() │     │ canBreathe() │
  └──────┬───────┘     └──────┬───────┘
         └──────┬──────────────┘
                │
        ┌───────┴───────┐
        │  Crocodile    │
        │  canBreathe() │ ← overrides both — no ambiguity!
        └───────────────┘
```

```java
interface WaterAnimal { boolean canBreathe(); }
interface LandAnimal  { boolean canBreathe(); }

class Crocodile implements WaterAnimal, LandAnimal {
    @Override
    public boolean canBreathe() { return true; }  // resolves diamond
}
```

## Methods in an Interface

| Rule | Detail |
|---|---|
| All methods are **implicitly `public`** | Even without modifier |
| All methods are **implicitly `abstract`** | Before Java 8 |
| Cannot be `final` | `final` prevents overriding, but interface methods MUST be overridden |

```java
interface Bird {
    void fly();                  // = public abstract void fly();
    public void hasBeak();       // same — public is redundant
}
```

## Fields in an Interface

All variables are **implicitly `public static final`** — i.e., **constants**.

```java
interface Bird {
    int MAX_HEIGHT = 2000;
    // = public static final int MAX_HEIGHT = 2000;
}
```

| Rule | Detail |
|---|---|
| Always `public` | Cannot be `private` or `protected` |
| Always `static final` | Constants — value cannot change |
| Must be initialized | At declaration time |

## Interface Implementation

### Rules

| Rule | Detail |
|---|---|
| Use `implements` keyword | `class Eagle implements Bird { }` |
| Cannot restrict access | If interface method is `public`, override must also be `public` |
| Concrete class | Must implement **all** abstract methods |
| Abstract class | Can implement some, leave rest abstract |
| Multiple interfaces | `class X implements A, B, C { }` ✅ |

```java
// Concrete class — must implement everything
class Eagle implements Bird {
    @Override
    public void fly() { System.out.println("Soaring"); }  // ✅
}

// Abstract class — partial implementation OK
abstract class AbstractEagle implements Bird {
    @Override
    public void fly() { ... }  // implemented
    // canRun() left abstract for child
}

class WhiteEagle extends AbstractEagle {
    // must implement remaining abstract methods
}
```

## Nested Interfaces

An interface can be declared **inside another interface** or **inside a class**.

### Interface Inside Interface

```java
interface Bird {
    void fly();                         // outer method
    
    public interface NonFlyingBird {    // nested — must be public
        void canRun();
    }
}

// Implement outer only
class Eagle implements Bird {
    @Override public void fly() { ... }
    // NOT required to implement NonFlyingBird
}

// Implement inner only
class Penguin implements Bird.NonFlyingBird {
    @Override public void canRun() { ... }
}

// Implement both
class Ostrich implements Bird, Bird.NonFlyingBird {
    @Override public void fly() { ... }
    @Override public void canRun() { ... }
}
```

### Interface Inside Class

```java
class Bird {
    protected interface NonFlyingBird {   // can be private/protected/public
        void canRun();
    }
}

class Penguin implements Bird.NonFlyingBird {
    @Override public void canRun() { ... }
}
```

| Location | Allowed Modifiers |
|---|---|
| Inside interface | `public` only |
| Inside class | `public`, `protected`, `private`, default |

## Interface vs Abstract Class

| Feature | Abstract Class | Interface |
|---|---|---|
| **Keyword** | `abstract class` | `interface` |
| **Child keyword** | `extends` | `implements` |
| **Methods** | Abstract + concrete | Abstract only (before Java 8) |
| **Java 8+** | — | `default`, `static` methods allowed |
| **Java 9+** | — | `private` methods allowed |
| **Variables** | Any modifier, any state | `public static final` (constants only) |
| **Member access** | `private`, `protected`, `public`, default | `public` only (Java 9: `private`) |
| **Constructor** | ✅ Yes | ❌ No |
| **Multiple inheritance** | ❌ Single class only | ✅ Multiple interfaces |
| **Extend** | One class + multiple interfaces | Multiple interfaces only |
| **Abstraction level** | 0–100% | 100% (before Java 8) |

## Summary

| Concept | Key Point |
|---|---|
| **Interface** | Contract — defines WHAT, not HOW |
| **Abstraction** | 100% — only signatures |
| **Polymorphism** | Interface as data type, runtime dispatch |
| **Multiple Inheritance** | Only via interfaces (diamond problem solved) |
| **Methods** | Implicitly `public abstract` |
| **Fields** | Implicitly `public static final` (constants) |
| **Implementation** | Concrete class must override all; abstract class can skip some |
| **Nested** | Interface in interface (public only) or in class (any modifier) |
