## What is a Stream?

A **pipeline** for processing collections of data in a functional style. Every stream pipeline has 3 steps:

```
  ┌──────────────┐     ┌─────────────────────┐     ┌──────────────────┐
  │ 1. CREATE     │ ──▶ │ 2. INTERMEDIATE OPS  │ ──▶ │ 3. TERMINAL OP   │
  │    Stream     │     │    (Lazy — chained)  │     │    (Triggers all)│
  └──────────────┘     └─────────────────────┘     └──────────────────┘
       Source             filter, map, sorted           forEach, collect
```

## Creating Streams

### From Collection

```java
List<Integer> list = List.of(1, 2, 3, 4, 5);
Stream<Integer> stream = list.stream();
```

### From Array

```java
int[] arr = {1, 2, 3};
IntStream intStream = Arrays.stream(arr);

String[] names = {"Alice", "Bob"};
Stream<String> stream = Arrays.stream(names);
```

### Using Stream.of()

```java
Stream<Integer> stream = Stream.of(1, 2, 3, 4, 5);
```

### Using Stream.builder()

```java
Stream<String> stream = Stream.<String>builder()
    .add("Alice")
    .add("Bob")
    .add("Charlie")
    .build();
```

### Using Stream.iterate()

```java
// Generate infinite stream — use limit() to bound it
Stream<Integer> stream = Stream.iterate(0, n -> n + 2).limit(5);
// → 0, 2, 4, 6, 8
```

## Intermediate Operations (Lazy)

These operations are **lazy** — they don't execute until a terminal operation is called. They return a new `Stream`.

### filter(Predicate)

```java
list.stream()
    .filter(n -> n > 3)    // keep elements where condition is true
    .forEach(System.out::println);  // 4, 5
```

### map(Function)

```java
list.stream()
    .map(n -> n * 2)       // transform each element
    .forEach(System.out::println);  // 2, 4, 6, 8, 10
```

### flatMap(Function)

Flattens nested collections into a single stream.

```java
List<List<Integer>> nested = List.of(List.of(1, 2), List.of(3, 4));
nested.stream()
    .flatMap(Collection::stream)    // flatten
    .forEach(System.out::println);  // 1, 2, 3, 4
```

### distinct()

```java
Stream.of(1, 2, 2, 3, 3, 3)
    .distinct()
    .forEach(System.out::println);  // 1, 2, 3
```

### sorted()

```java
// Natural ordering
Stream.of(3, 1, 2).sorted().forEach(System.out::println);  // 1, 2, 3

// Custom comparator
Stream.of(3, 1, 2).sorted(Comparator.reverseOrder())
    .forEach(System.out::println);  // 3, 2, 1
```

### peek(Consumer)

For debugging — perform an action without modifying the stream.

```java
list.stream()
    .peek(n -> System.out.println("Before filter: " + n))
    .filter(n -> n > 3)
    .peek(n -> System.out.println("After filter: " + n))
    .forEach(System.out::println);
```

### limit(n) & skip(n)

```java
Stream.of(1, 2, 3, 4, 5)
    .skip(2)       // skip first 2 → [3, 4, 5]
    .limit(2)      // take next 2  → [3, 4]
    .forEach(System.out::println);  // 3, 4
```

### mapToInt / mapToLong / mapToDouble

Convert to primitive streams for numeric operations.

```java
int sum = list.stream()
    .mapToInt(Integer::intValue)
    .sum();                          // 15
```

### All Intermediate Operations

| Operation | Parameter | Purpose |
|---|---|---|
| `filter` | `Predicate<T>` | Keep matching elements |
| `map` | `Function<T, R>` | Transform each element |
| `flatMap` | `Function<T, Stream<R>>` | Flatten nested streams |
| `distinct` | — | Remove duplicates |
| `sorted` | `Comparator<T>` (optional) | Sort elements |
| `peek` | `Consumer<T>` | Debug/side-effect (no modification) |
| `limit` | `long n` | Take first n elements |
| `skip` | `long n` | Skip first n elements |
| `mapToInt` | `ToIntFunction<T>` | Convert to IntStream |

## Terminal Operations (Trigger Execution)

Terminal operations **trigger** the entire pipeline and produce a result. After a terminal operation, the stream is consumed.

### forEach(Consumer)

```java
list.stream().forEach(System.out::println);
```

### toArray()

```java
Object[] arr = list.stream().toArray();
Integer[] arr2 = list.stream().toArray(Integer[]::new);
```

### reduce(BinaryOperator)

Aggregates all elements into a single value.

```java
// Sum
int sum = Stream.of(1, 2, 3, 4)
    .reduce(0, (a, b) -> a + b);    // 10

// Max
int max = Stream.of(1, 2, 3, 4)
    .reduce(Integer.MIN_VALUE, Integer::max);  // 4

// Concatenate strings
String result = Stream.of("A", "B", "C")
    .reduce("", (a, b) -> a + b);   // "ABC"
```

### collect(Collector)

```java
List<Integer> result = list.stream()
    .filter(n -> n > 2)
    .collect(Collectors.toList());   // [3, 4, 5]

Set<Integer> set = list.stream()
    .collect(Collectors.toSet());

String joined = Stream.of("A", "B", "C")
    .collect(Collectors.joining(", "));  // "A, B, C"
```

### min / max

```java
Optional<Integer> min = list.stream().min(Comparator.naturalOrder());  // 1
Optional<Integer> max = list.stream().max(Comparator.naturalOrder());  // 5
```

### count()

```java
long count = list.stream().filter(n -> n > 3).count();  // 2
```

### Match Operations

```java
boolean anyEven  = list.stream().anyMatch(n -> n % 2 == 0);   // true (at least one)
boolean allEven  = list.stream().allMatch(n -> n % 2 == 0);   // false (not all)
boolean noneNeg  = list.stream().noneMatch(n -> n < 0);       // true (none negative)
```

### find Operations

```java
Optional<Integer> first = list.stream().findFirst();   // Optional[1]
Optional<Integer> any   = list.stream().findAny();     // Optional[1] (non-deterministic in parallel)
```

### All Terminal Operations

| Operation | Returns | Purpose |
|---|---|---|
| `forEach` | void | Perform action on each element |
| `toArray` | `Object[]` or `T[]` | Collect to array |
| `reduce` | `T` or `Optional<T>` | Aggregate to single value |
| `collect` | `R` | Collect to collection/string |
| `min` / `max` | `Optional<T>` | Find min/max with comparator |
| `count` | `long` | Count elements |
| `anyMatch` | `boolean` | At least one matches? |
| `allMatch` | `boolean` | All match? |
| `noneMatch` | `boolean` | None match? |
| `findFirst` | `Optional<T>` | First element |
| `findAny` | `Optional<T>` | Any element (parallel-friendly) |

## Lazy Evaluation & Processing Order

### Lazy = Nothing Happens Until Terminal Op

```java
Stream<Integer> stream = list.stream()
    .filter(n -> {
        System.out.println("Filtering: " + n);  // NOT printed yet!
        return n > 3;
    });
// No output — filter hasn't executed

stream.forEach(System.out::println);  // NOW it executes
```

### Processing Order: Element-by-Element (NOT Operation-by-Operation)

```
  Elements: [1, 2, 3]
  Pipeline: filter → map → forEach

  ❌ NOT this (operation-by-operation):
     filter ALL → [2, 3] → map ALL → [4, 6] → forEach ALL

  ✅ THIS (element-by-element):
     1 → filter(1) → skip
     2 → filter(2) → pass → map(2) → 4 → forEach(4)
     3 → filter(3) → pass → map(3) → 6 → forEach(6)
```

> **Exception:** Stateful operations like `sorted()` must collect ALL elements before proceeding.

```
  filter → sorted → forEach
  1 → filter → pass → sorted (WAIT for all elements)
  2 → filter → skip
  3 → filter → pass → sorted (WAIT)
  sorted collects: [1, 3] → sorts → [1, 3]
  1 → forEach
  3 → forEach
```

## Stream Can Only Be Used Once

After a terminal operation, the stream is **consumed** and cannot be reused.

```java
Stream<Integer> stream = list.stream();
stream.forEach(System.out::println);   // ✅ first use
stream.forEach(System.out::println);   // ❌ IllegalStateException!
```

> **Solution:** Create a new stream each time.

## Parallel Streams

Process elements concurrently using multiple threads for **better performance on large datasets**.

```java
// Create parallel stream
List<Integer> list = List.of(1, 2, 3, 4, 5, 6, 7, 8);
list.parallelStream()
    .filter(n -> n > 3)
    .forEach(System.out::println);  // order may vary!
```

### How It Works Internally

```
  Data: [1, 2, 3, 4, 5, 6, 7, 8]
              │
     ┌────────┴────────┐          Spliterator.trySplit()
     │                 │
  [1,2,3,4]       [5,6,7,8]
     │                 │          Fork-Join Pool
  Thread 1          Thread 2
  filter+map       filter+map
     │                 │
     └────────┬────────┘
              │                   Join results
        Combined Result
```

| Feature | Sequential Stream | Parallel Stream |
|---|---|---|
| Method | `.stream()` | `.parallelStream()` |
| Threads | Single (main thread) | Multiple (fork-join pool) |
| Order | Guaranteed | NOT guaranteed (use `forEachOrdered` if needed) |
| Best for | Small data, order-dependent ops | Large datasets, independent operations |
| Overhead | None | Thread management overhead |

> **When to use parallel?** Large collections with independent, stateless operations. Avoid for small datasets — thread overhead outweighs gains.

## Summary

| Concept | Key Point |
|---|---|
| **Stream** | Pipeline: create → intermediate ops → terminal op |
| **Intermediate** | Lazy — `filter`, `map`, `flatMap`, `sorted`, `distinct`, `peek`, `limit`, `skip` |
| **Terminal** | Triggers execution — `forEach`, `collect`, `reduce`, `count`, `min`/`max`, `findFirst` |
| **Lazy evaluation** | Nothing runs until terminal op is invoked |
| **Processing order** | Element-by-element through pipeline (except stateful ops like `sorted`) |
| **Single use** | Stream consumed after terminal op — create new for reuse |
| **Parallel** | `parallelStream()` — fork-join pool — faster for large data |
