## Queue Interface

Queue follows **FIFO** (First In, First Out). Provides 6 key methods in 3 pairs:

### Methods (3 Pairs)

| Operation | Throws Exception | Returns Special Value |
|---|---|---|
| **Insert** | `add(e)` — throws `IllegalStateException` if full | `offer(e)` — returns `false` if full |
| **Remove head** | `remove()` — throws `NoSuchElementException` if empty | `poll()` — returns `null` if empty |
| **Examine head** | `element()` — throws `NoSuchElementException` if empty | `peek()` — returns `null` if empty |

```java
Queue<Integer> queue = new LinkedList<>();
queue.offer(10);       // [10]
queue.offer(20);       // [10, 20]
queue.peek();          // 10 (head — doesn't remove)
queue.poll();          // 10 (removes head) → [20]
```

## PriorityQueue

Elements are ordered by **priority**, not insertion order. Internally uses a **priority heap**.

### Min Heap (Default — Natural Ordering)

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.add(30);
minHeap.add(10);
minHeap.add(20);
minHeap.poll();  // 10 ← smallest first
minHeap.poll();  // 20
minHeap.poll();  // 30
```

### Max Heap (Using Comparator)

```java
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
// OR: new PriorityQueue<>((a, b) -> b - a);
maxHeap.add(30);
maxHeap.add(10);
maxHeap.add(20);
maxHeap.poll();  // 30 ← largest first
```

### Time Complexity

| Operation | Time |
|---|---|
| `add()` / `offer()` | O(log n) |
| `peek()` | O(1) |
| `poll()` | O(log n) |
| `remove(Object)` | O(n) |

## Comparator

A **functional interface** that defines custom comparison logic for sorting objects.

```java
@FunctionalInterface
public interface Comparator<T> {
    int compare(T o1, T o2);    // the one abstract method
}
```

### Return Value Rules

```
  compare(o1, o2)
  ┌────────────────────────────────────────────────┐
  │ Returns > 0  (positive)  →  o1 > o2  → SWAP   │
  │ Returns 0    (zero)      →  o1 = o2  → NO SWAP│
  │ Returns < 0  (negative)  →  o1 < o2  → NO SWAP│
  └────────────────────────────────────────────────┘
```

### Ascending vs Descending

```java
// Ascending order:  o1 - o2
Comparator<Integer> ascending  = (v1, v2) -> v1 - v2;

// Descending order: o2 - o1
Comparator<Integer> descending = (v1, v2) -> v2 - v1;
```

```
  Example: values = [30, 10, 20]

  Ascending (v1 - v2):
    compare(30, 10) = 30 - 10 = +20 → positive → SWAP → [10, 30, 20]
    compare(30, 20) = 30 - 20 = +10 → positive → SWAP → [10, 20, 30] ✅

  Descending (v2 - v1):
    compare(30, 10) = 10 - 30 = -20 → negative → NO SWAP → [30, 10, 20]
    compare(10, 20) = 20 - 10 = +10 → positive → SWAP → [30, 20, 10] ✅
```

### Sorting Custom Objects

```java
class Car {
    String type;     // "SUV", "Sedan"
    String fuel;     // "Petrol", "CNG"
    int price;
}

// Sort by price (ascending)
Comparator<Car> byPrice = (c1, c2) -> c1.price - c2.price;

// Sort by type (alphabetical)
Comparator<Car> byType = (c1, c2) -> c1.type.compareTo(c2.type);

List<Car> cars = new ArrayList<>();
cars.add(new Car("SUV", "Petrol", 50000));
cars.add(new Car("Sedan", "CNG", 30000));

Collections.sort(cars, byPrice);   // sorted by price
Collections.sort(cars, byType);    // sorted by type
```

> **Key advantage:** Multiple `Comparator` implementations for different sort strategies — without modifying the class.

## Sorting Primitives vs Objects

| Feature | Primitives | Objects |
|---|---|---|
| `Arrays.sort(arr)` | ✅ Works directly | ❌ Needs `Comparator` or `Comparable` |
| Default order | Natural ordering (ascending) | No natural ordering → `ClassCastException` |
| Internal algorithm | Dual-Pivot QuickSort | TimSort (merge sort variant) |

```java
// Primitive — works out of the box
int[] nums = {3, 1, 2};
Arrays.sort(nums);  // [1, 2, 3] ✅

// Object — needs Comparator or Comparable
Car[] cars = { ... };
Arrays.sort(cars);                        // ❌ ClassCastException!
Arrays.sort(cars, (c1, c2) -> c1.price - c2.price);  // ✅
```

## Comparable

An interface **implemented by the object itself** to define its **natural ordering**.

```java
public interface Comparable<T> {
    int compareTo(T other);     // compare THIS object with OTHER
}
```

### Implementation

```java
class Car implements Comparable<Car> {
    String type;
    String fuel;
    int price;

    @Override
    public int compareTo(Car other) {
        return this.type.compareTo(other.type);  // sort by type (ascending)
    }
}

// Now sorting works WITHOUT explicit comparator
List<Car> cars = new ArrayList<>();
cars.add(new Car("SUV", "Petrol", 50000));
cars.add(new Car("Sedan", "CNG", 30000));
Collections.sort(cars);  // ✅ uses compareTo() internally
```

### How It Works Internally

```
  Collections.sort(cars)
      → list.sort()
          → Arrays.sort()
              → comparator == null?  YES
                  → use natural ordering (compareTo)
                      → car1.compareTo(car2)
                          → "SUV".compareTo("Sedan")
                              → positive → SWAP
```

## Comparator vs Comparable

| Feature | Comparator | Comparable |
|---|---|---|
| **Package** | `java.util` | `java.lang` |
| **Method** | `compare(o1, o2)` | `compareTo(other)` |
| **Parameters** | Two objects | One object (compares with `this`) |
| **Implementation** | External class / lambda | Inside the class being sorted |
| **Sort strategies** | ✅ Multiple (different comparators) | ❌ Single natural ordering |
| **Class modification** | ❌ Not needed | ✅ Class must implement |
| **Flexibility** | Sort by price, by type, by fuel — any field | Only one sort order |
| **Use case** | `Collections.sort(list, comparator)` | `Collections.sort(list)` |
| **Type** | Functional interface (Java 8) | Regular interface |

```java
// Comparable — ONE sort order, defined inside the class
class Car implements Comparable<Car> {
    public int compareTo(Car other) {
        return this.price - other.price;    // always by price
    }
}
Collections.sort(cars);                     // by price (fixed)

// Comparator — MULTIPLE sort orders, defined externally
Collections.sort(cars, (c1, c2) -> c1.type.compareTo(c2.type));   // by type
Collections.sort(cars, (c1, c2) -> c1.price - c2.price);          // by price
Collections.sort(cars, (c1, c2) -> c1.fuel.compareTo(c2.fuel));   // by fuel
```

## Summary

| Concept | Key Point |
|---|---|
| **Queue** | FIFO — `offer/poll/peek` (safe) vs `add/remove/element` (throw) |
| **PriorityQueue** | Min-heap default, O(log n) insert/poll, O(1) peek |
| **Comparator** | External comparison — `compare(o1, o2)` — multiple strategies |
| **Comparable** | Internal comparison — `compareTo(other)` — single natural order |
| **Ascending** | `o1 - o2` (positive → swap) |
| **Descending** | `o2 - o1` (positive → swap) |
