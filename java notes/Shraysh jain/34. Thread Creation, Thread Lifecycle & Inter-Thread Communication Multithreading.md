## Two Ways to Create a Thread

### Why Two Ways?

Java allows only **single class inheritance**. If a class already extends another class, it can't also extend `Thread`. But it can implement `Runnable` (multiple interfaces allowed).

### Way 1: Implement `Runnable` Interface (Preferred)

```java
// Step 1: Create class implementing Runnable
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("Running in: " + Thread.currentThread().getName());
    }
}

// Step 2: Create Runnable object
MyTask task = new MyTask();

// Step 3: Pass to Thread constructor
Thread thread = new Thread(task);

// Step 4: Start the thread
thread.start();  // internally calls run() → task.run()
```

```
  thread.start()
      → Thread.run()
          → if (target != null) target.run()   ← target = our Runnable
              → MyTask.run()                    ← our code executes
```

**Using Lambda (shorter):**

```java
Thread t = new Thread(() -> {
    System.out.println("Running in: " + Thread.currentThread().getName());
});
t.start();
```

### Way 2: Extend `Thread` Class

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Running in: " + Thread.currentThread().getName());
    }
}

MyThread t = new MyThread();
t.start();  // internally calls this.run() → MyThread.run()
```

### Comparison

| Feature | Runnable (Interface) | Thread (Class) |
|---|---|---|
| Inheritance | ✅ Can still extend another class | ❌ Can't extend another class |
| Separation of concerns | ✅ Task logic separate from thread | ❌ Mixed |
| Industry preference | ✅ Preferred | ❌ Rarely used |
| Reusability | ✅ Same Runnable for multiple threads | ❌ Less reusable |

## Thread Lifecycle (States)

```
                         ┌──────────┐
        new Thread()     │   NEW    │
                         └────┬─────┘
                              │ start()
                         ┌────▼─────┐
                    ┌────│ RUNNABLE │◄────────────────────────┐
                    │    │(Ready +  │                         │
                    │    │ Running) │                         │
                    │    └──┬───┬───┘                         │
                    │       │   │                             │
         ┌──────────┤       │   ├──────────────┐             │
         │          │       │   │              │             │
    ┌────▼────┐  ┌──▼──────▼┐  ┌─────▼──────┐  │
    │ BLOCKED │  │ WAITING  │  │TIME_WAITING│  │
    │         │  │          │  │            │  │
    │ IO/Lock │  │ wait()   │  │ sleep(ms)  │  │
    │ acquire │  │          │  │ join(ms)   │  │
    └────┬────┘  └────┬─────┘  └─────┬──────┘  │
         │            │              │          │
         │  IO done / │  notify() /  │ timeout  │
         │  lock free │  notifyAll() │ expires  │
         └────────────┴──────────────┴──────────┘
                              │
                    ┌─────────▼────────┐
                    │   TERMINATED     │
                    │ (dead — no restart)│
                    └──────────────────┘
```

| State | Description | Monitor Lock |
|---|---|---|
| **NEW** | Thread created, not started | — |
| **RUNNABLE** | Ready to run / running (waiting for CPU or got CPU) | Acquires lock |
| **BLOCKED** | Waiting for IO, or lock held by another thread | Releases monitor lock ✅ |
| **WAITING** | `wait()` called — indefinite wait | Releases monitor lock ✅ |
| **TIMED_WAITING** | `sleep(ms)` / `join(ms)` — wait for specific time | Does NOT release lock ❌ |
| **TERMINATED** | Execution complete — cannot restart | — |

## Monitor Lock & Synchronized

Every Java object has a **monitor lock**. `synchronized` uses it to allow only one thread into a critical section.

```java
class SharedResource {
    synchronized void task() {    // lock on 'this' object
        // only ONE thread at a time
    }
}
```

### Synchronized Block

```java
void task() {
    System.out.println("Before sync");   // no lock needed
    synchronized (this) {                 // lock acquired here
        // critical section
    }                                     // lock released here
}
```

### Key Rules

| Rule | Detail |
|---|---|
| Lock is per **object** | Same object → shared lock; different objects → different locks |
| Same object, different methods | Both `synchronized` → still share the same lock |
| Non-synchronized method | Any thread can enter — no lock needed |
| `sleep()` | Does NOT release monitor lock |
| `wait()` | Releases monitor lock |

```java
SharedResource obj = new SharedResource();

// Thread 1: obj.task1() ← acquires lock on obj
// Thread 2: obj.task2() ← WAITS (same obj, same lock)
// Thread 3: obj.task3() ← runs freely (not synchronized)
```

## Inter-Thread Communication

### wait(), notify(), notifyAll()

| Method | Effect |
|---|---|
| `wait()` | Current thread releases lock, goes to WAITING state |
| `notify()` | Wakes up ONE arbitrary thread waiting on the same object's lock |
| `notifyAll()` | Wakes up ALL threads waiting on the same object's lock |

> Must be called inside a `synchronized` block/method — they operate on the monitor lock.

### Spurious Wakeup

Threads can wake up without `notify()` due to system noise. Always use `while` loop, not `if`:

```java
synchronized (resource) {
    while (!conditionMet) {    // ✅ while — re-checks after wakeup
        resource.wait();
    }
    // proceed
}
```

## Producer-Consumer Example

```java
class SharedResource {
    private Queue<Integer> queue = new LinkedList<>();
    private int bufferSize;

    SharedResource(int size) { this.bufferSize = size; }

    synchronized void produce(int item) throws InterruptedException {
        while (queue.size() == bufferSize) {
            wait();                              // buffer full — wait
        }
        queue.add(item);
        System.out.println("Produced: " + item);
        notify();                                // wake consumer
    }

    synchronized void consume() throws InterruptedException {
        while (queue.isEmpty()) {
            wait();                              // buffer empty — wait
        }
        int item = queue.poll();
        System.out.println("Consumed: " + item);
        notify();                                // wake producer
    }
}
```

```
  Producer Thread              SharedResource              Consumer Thread
       │                     ┌──────────────┐                    │
       │── produce(item) ──▶ │ Queue [_, _] │ ◄── consume() ────│
       │                     │ bufferSize=3 │                    │
       │   if full → wait()  │              │  if empty → wait() │
       │   add → notify()    │              │  poll → notify()   │
       └─────────────────────┴──────────────┴────────────────────┘
```

```java
// Main
SharedResource resource = new SharedResource(3);

Thread producer = new Thread(() -> {
    for (int i = 1; i <= 6; i++) resource.produce(i);
});

Thread consumer = new Thread(() -> {
    for (int i = 1; i <= 6; i++) resource.consume();
});

producer.start();
consumer.start();
```

## Summary

| Concept | Key Point |
|---|---|
| **Runnable** | Preferred — `implements Runnable`, pass to `new Thread(runnable)` |
| **Extending Thread** | Less flexible — class can't extend anything else |
| **Lifecycle** | NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED |
| **Monitor lock** | Per-object lock — `synchronized` acquires it |
| **wait()** | Releases lock, thread goes to WAITING |
| **notify()** | Wakes one waiting thread on same object |
| **sleep()** | Does NOT release lock — thread holds it |
| **Producer-Consumer** | `wait()` when full/empty, `notify()` after produce/consume |
