## What is an Exception?

An **event** that disrupts the normal flow of a program at runtime.

```
  Normal Flow:  statement1 → statement2 → statement3 → statement4
                                  ↓ exception!
  Disrupted:    statement1 → statement2 ──▶ program terminates
```

When an exception occurs, Java creates an **exception object** and hands it off to the runtime system (call stack propagation).

---

## Exception Object

Every exception in Java is an **object** with three key properties:

```
  ┌──────────────────────────────────┐
  │         Exception Object          │
  ├──────────────────────────────────┤
  │  Type:    ArithmeticException    │
  │  Message: "/ by zero"            │
  │  Stack Trace:                    │
  │    at Main.method3(Main.java:20) │
  │    at Main.method2(Main.java:15) │
  │    at Main.method1(Main.java:10) │
  │    at Main.main(Main.java:5)     │
  └──────────────────────────────────┘
```

| Property | Description |
|---|---|
| **Type** | Class of exception (`ArithmeticException`, `NullPointerException`) |
| **Message** | Human-readable description of what went wrong |
| **Stack Trace** | Call chain from where exception occurred back to `main` |

## Exception Hierarchy

```
                    Object
                      │
                  Throwable
                 ╱         ╲
             Error        Exception
             │              │
     ┌───────┴──────┐      ├── RuntimeException (Unchecked)
     │              │      │     ├── ArithmeticException
  OutOfMemory  StackOverflow │     ├── NullPointerException
  Error        Error       │     ├── ClassCastException
                           │     ├── ArrayIndexOutOfBoundsException
                           │     ├── StringIndexOutOfBoundsException
                           │     └── NumberFormatException
                           │
                           └── Checked Exceptions
                                 ├── ClassNotFoundException
                                 ├── InterruptedException
                                 ├── IOException
                                 └── FileNotFoundException
```

## Error vs Exception

| Feature | Error | Exception |
|---|---|---|
| **Source** | JVM / system | Application code |
| **Controllable** | ❌ Cannot handle | ✅ Can catch and recover |
| **Examples** | `OutOfMemoryError`, `StackOverflowError` | `ArithmeticException`, `IOException` |
| **Should catch?** | No — nothing you can do | Yes — handle gracefully |

---

## Checked vs Unchecked Exceptions

| Feature | Checked (Compile-time) | Unchecked (Runtime) |
|---|---|---|
| **Compiler forces handling** | ✅ Must handle or declare `throws` | ❌ No compiler enforcement |
| **Parent class** | `Exception` (not RuntimeException) | `RuntimeException` |
| **When detected** | At compile time | At runtime |
| **Common examples** | `IOException`, `ClassNotFoundException`, `InterruptedException` | `ArithmeticException`, `NullPointerException`, `ClassCastException` |

### Common Runtime Exceptions

```java
// ClassCastException
Object obj = "hello";
Integer num = (Integer) obj;           // ❌ String → Integer

// ArithmeticException
int result = 10 / 0;                   // ❌ divide by zero

// ArrayIndexOutOfBoundsException
int[] arr = {1, 2, 3};
int val = arr[5];                      // ❌ index 5 doesn't exist

// NullPointerException
String s = null;
s.length();                            // ❌ null reference

// NumberFormatException
int num = Integer.parseInt("abc");     // ❌ "abc" is not a number
```

### Common Checked Exceptions

```java
// ClassNotFoundException
Class.forName("com.unknown.ClassName"); // must handle

// FileNotFoundException
new FileInputStream("nonexistent.txt"); // must handle

// InterruptedException
Thread.sleep(1000);                     // must handle
```

## 5 Keywords

```
  ┌────────────────────────────────────────────────────────┐
  │  try      → wrap risky code                            │
  │  catch    → handle specific exception                  │
  │  finally  → always execute (cleanup)                   │
  │  throw    → manually throw an exception                │
  │  throws   → delegate exception to caller               │
  └────────────────────────────────────────────────────────┘
```

## try-catch Block

```java
try {
    int result = 10 / 0;              // risky code
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero: " + e.getMessage());
}
// program continues normally after catch
```

### Call Stack Propagation

```
  main() → method1() → method2() → method3()
                                        ↓ exception!
           Does method3 have try-catch?
               No → propagate to method2
           Does method2 have try-catch?
               No → propagate to method1
           Does method1 have try-catch?
               Yes → handle in catch block ✅
               No → propagate to main → terminate ❌
```

## Multiple Catch & Multi-Catch

### Multiple Catch Blocks

**Order matters** — specific exceptions FIRST, generic LAST.

```java
try {
    // risky code
} catch (ArithmeticException e) {      // specific FIRST
    System.out.println("Math error");
} catch (NullPointerException e) {     // specific
    System.out.println("Null reference");
} catch (Exception e) {                // generic LAST — catches everything else
    System.out.println("Something went wrong");
}
```

```java
// ❌ WRONG — generic first catches everything, specific is unreachable
try { ... }
catch (Exception e) { ... }            // catches ALL → code below unreachable
catch (ArithmeticException e) { ... }  // ❌ Compilation Error
```

### Multi-Catch with Pipe (Java 7)

```java
try {
    // risky code
} catch (ArithmeticException | NullPointerException e) {   // pipe = OR
    System.out.println("Error: " + e.getMessage());
}
```

| Rule | Detail |
|---|---|
| Exceptions in `\|` must NOT be related | `IOException \| FileNotFoundException` ❌ (parent-child) |
| Single variable | `e` is shared across all types in the multi-catch |

## finally Block

Code that **always executes**, regardless of whether an exception occurred.

```java
try {
    int result = 10 / 2;
    System.out.println("Success");
} catch (ArithmeticException e) {
    System.out.println("Error");
} finally {
    System.out.println("Cleanup — always runs");  // ALWAYS
}
```

### When Does `finally` NOT Execute?

| Scenario | `finally` runs? |
|---|---|
| No exception | ✅ Yes |
| Exception caught | ✅ Yes |
| Exception NOT caught | ✅ Yes (before termination) |
| `System.exit(0)` called | ❌ No |
| JVM crashes / `Error` | ❌ No |

### try-finally Without catch

```java
try {
    // risky code
} finally {
    // cleanup — no catch needed
}
// If exception occurs, finally runs → exception still propagates
```

## throw & throws

### `throw` — Create & Throw Exception

```java
if (name.equals("dummy")) {
    throw new IllegalArgumentException("Invalid name: " + name);
}
```

### `throws` — Delegate to Caller

```java
void method1() throws ClassNotFoundException {
    Class.forName("com.unknown.ClassName");  // don't handle here
    // delegate to whoever calls method1()
}

void main() {
    try {
        method1();
    } catch (ClassNotFoundException e) {
        System.out.println("Class not found: " + e.getMessage());
    }
}
```

### Rethrow Pattern

```java
void main() {
    try {
        method1();
    } catch (ClassNotFoundException e) {
        log.error("Logging before rethrow", e);   // log first
        throw e;                                    // then rethrow
    }
}
```

### throw vs throws

| Feature | `throw` | `throws` |
|---|---|---|
| Purpose | Create & throw an exception | Declare: "this method may throw" |
| Location | Inside method body | Method signature |
| Quantity | One exception at a time | Multiple: `throws A, B, C` |
| Forces handling? | No | Yes (for checked exceptions) |

## Custom Exceptions

### Creating a Custom Exception

```java
public class MyCustomException extends Exception {     // checked
    public MyCustomException(String message) {
        super(message);          // pass message to Exception
    }
}

public class MyRuntimeException extends RuntimeException {  // unchecked
    public MyRuntimeException(String message) {
        super(message);
    }
}
```

### Using a Custom Exception

```java
void method1() throws MyCustomException {
    throw new MyCustomException("Some issue arose");
}

void main() {
    try {
        method1();
    } catch (MyCustomException e) {
        System.out.println(e.getMessage());  // "Some issue arose"
    }
}
```

> If your custom exception extends `Exception` → checked → must be handled or declared with `throws`.
> If it extends `RuntimeException` → unchecked → no compiler enforcement.

## Why Exception Handling?

### Advantages

| Advantage | Detail |
|---|---|
| **Clean code** | Separates error-handling from regular logic (try-catch vs. if-else error codes) |
| **Error propagation** | Exception objects carry type + message + stack trace (no manual error codes) |
| **Recovery** | Catch and recover — program can continue |
| **Debugging** | Stack trace pinpoints exact line of failure |
| **Security** | Hide sensitive info in logs (mask PII in custom error messages) |

### Without Exception Handling

```java
// ❌ Messy — if-else validation mixed with business logic
int result = getStudentCapacity(classNumber);
if (result == 0) return -2;
String[] names = new String[result];
if (names == null || names.length == 0) return -3;
names[0] = "NewValue";
```

### With Exception Handling

```java
// ✅ Clean — business logic in try, errors in catch
try {
    int result = getStudentCapacity(classNumber);
    String[] names = new String[result];
    names[0] = "NewValue";
} catch (IndexOutOfBoundsException e) {
    // handle
} catch (Exception e) {
    // handle
}
```

### Disadvantage

Exception handling is **slightly expensive** — if the stack trace is deep and exceptions are unhandled, the JVM must traverse the entire call stack.

> **Tip:** Where a simple `if` check can prevent the exception, prefer the `if` check:
> ```java
> // ✅ Prefer this
> if (b != 0) return a / b; else return -1;
>
> // ❌ Over this
> try { return a / b; } catch (ArithmeticException e) { return -1; }
> ```

## Summary

| Concept | Key Point |
|---|---|
| **Exception** | Event disrupting normal flow → exception object (type, message, stack trace) |
| **Error** | JVM-level, unrecoverable (`OutOfMemoryError`) |
| **Checked** | Compiler forces handling (`IOException`) |
| **Unchecked** | Runtime only (`NullPointerException`) |
| **try-catch** | Handle exception — program continues |
| **finally** | Always runs (cleanup) — except `System.exit` / JVM crash |
| **throw** | Create and throw exception |
| **throws** | Declare exception in method signature |
| **Custom** | Extend `Exception` (checked) or `RuntimeException` (unchecked) |
