## What is a Functional Interface?

An interface with **exactly one abstract method**. Also called **SAM** (Single Abstract Method) interface.

```java
@FunctionalInterface          // optional but recommended
interface Bird {
    boolean canFly(String type);  // THE one abstract method
    
    default void eat() { }        // ✅ allowed
    static void info() { }        // ✅ allowed
    String toString();            // ✅ Object class method — not counted
}
```

### Rules

| Rule | Detail |
|---|---|
| Exactly **one** abstract method | More → compilation error (if annotated) |
| `default` methods allowed | ✅ Any number |
| `static` methods allowed | ✅ Any number |
| `Object` class methods allowed | ✅ `toString()`, `equals()`, `hashCode()` don't count |
| `@FunctionalInterface` annotation | Optional — but **enforces** single-abstract restriction |

```java
@FunctionalInterface
interface Bird {
    void fly();
    void eat();   // ❌ Compilation Error — two abstract methods!
}
```

## Lambda Expression

A concise way to **implement a functional interface** — reduces boilerplate compared to anonymous classes.

### Syntax

```
(parameters) -> { body }
(parameters) -> expression       // single line — no braces, implicit return
```

### Example

```java
@FunctionalInterface
interface Bird {
    boolean canFly(String type);
}

// Lambda implementation
Bird eagle = (type) -> {
    return type.equals("Eagle");
};

// Single-line — no braces, no return keyword
Bird sparrow = (type) -> type.equals("Sparrow");

eagle.canFly("Eagle");    // true
sparrow.canFly("Hawk");   // false
```

## Three Ways to Implement

### 1. Traditional Class (`implements`)

```java
class Eagle implements Bird {
    @Override
    public boolean canFly(String type) {
        return true;
    }
}
Bird b = new Eagle();
```

### 2. Anonymous Class

```java
Bird b = new Bird() {
    @Override
    public boolean canFly(String type) {
        return true;
    }
};
```

### 3. Lambda Expression (Java 8) — Least Verbose

```java
Bird b = (type) -> true;
```

```
  Traditional:  class + implements + @Override + method name + body
  Anonymous:    new Interface() { @Override + method name + body }
  Lambda:       (params) -> body
                ↑ No class, no method name, no @Override
```

> **Why lambda only with functional interfaces?** Since there's exactly one abstract method, Java knows which method the lambda implements — no need to specify the name.

## Built-in Functional Interfaces

Java provides four core functional interfaces in `java.util.function`:

### 1. Consumer — Accept, No Return

```java
@FunctionalInterface
interface Consumer<T> {
    void accept(T t);
}
```

```java
Consumer<Integer> logger = (value) -> {
    if (value > 10) System.out.println("Logging: " + value);
};
logger.accept(11);  // "Logging: 11"
```

### 2. Supplier — No Input, Returns Result

```java
@FunctionalInterface
interface Supplier<T> {
    T get();
}
```

```java
Supplier<String> greeting = () -> "Hello World";
greeting.get();  // "Hello World"
```

### 3. Function — Accept Input, Return Result

```java
@FunctionalInterface
interface Function<T, R> {
    R apply(T t);
}
```

```java
Function<Integer, String> converter = (num) -> num.toString();
converter.apply(42);  // "42"
```

### 4. Predicate — Accept Input, Return Boolean

```java
@FunctionalInterface
interface Predicate<T> {
    boolean test(T t);
}
```

```java
Predicate<Integer> isEven = (num) -> num % 2 == 0;
isEven.test(4);   // true
isEven.test(7);   // false
```

### Quick Reference

| Interface | Input | Output | Method | Use Case |
|---|---|---|---|---|
| `Consumer<T>` | ✅ one | ❌ void | `accept(T)` | Logging, printing |
| `Supplier<T>` | ❌ none | ✅ T | `get()` | Factory, lazy init |
| `Function<T,R>` | ✅ one (T) | ✅ R | `apply(T)` | Transform/convert |
| `Predicate<T>` | ✅ one (T) | ✅ boolean | `test(T)` | Filter, validate |

> If these four don't fit your needs, create your own `@FunctionalInterface`.

## Inheritance Rules

### Functional → extends Non-Functional

```java
interface LivingThing { void canBreathe(); }     // non-functional (1 abstract)

@FunctionalInterface
interface Bird extends LivingThing {
    void fly();                                    // ❌ now 2 abstract methods!
}

// FIX: parent must use default
interface LivingThing { default void canBreathe() { } }

@FunctionalInterface
interface Bird extends LivingThing {
    void fly();                                    // ✅ only 1 abstract
}
```

### Non-Functional → extends Functional

```java
@FunctionalInterface
interface LivingThing { void canBreathe(); }

interface Bird extends LivingThing {              // NOT @FunctionalInterface
    void fly();                                    // ✅ 2 abstracts OK — not functional
}
```

### Functional → extends Functional

```java
@FunctionalInterface
interface LivingThing { boolean canBreathe(); }

@FunctionalInterface
interface Bird extends LivingThing {
    boolean canBreathe();  // ✅ SAME method — overrides, still 1 abstract
    // void fly();         // ❌ DIFFERENT method — would be 2 abstracts
}
```

## Summary

| Concept | Key Point |
|---|---|
| **Functional Interface** | Exactly one abstract method |
| **@FunctionalInterface** | Optional annotation — enforces the rule |
| **Lambda Expression** | `(params) -> body` — concise implementation |
| **Why lambda + functional?** | One method → no name needed → less verbose |
| **Consumer** | Takes input, returns nothing |
| **Supplier** | No input, returns result |
| **Function** | Takes input, returns result |
| **Predicate** | Takes input, returns boolean |
