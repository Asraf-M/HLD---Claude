## What are Meta-Annotations?

Annotations placed **on other annotations** to define their behavior — where they apply, how long they live, etc.

```
  @Target(ElementType.METHOD)         ← meta-annotation
  @Retention(RetentionPolicy.RUNTIME) ← meta-annotation
  public @interface MyAnnotation {    ← custom annotation
      String value();
  }
```

---

## 1. @Target

Specifies **where** the annotation can be applied.

```java
@Target(ElementType.METHOD)           // only on methods
public @interface MyAnnotation { }

@Target({ElementType.METHOD, ElementType.FIELD})  // methods AND fields
public @interface AnotherAnnotation { }
```

### ElementType Values

| ElementType | Can Apply To | Example |
|---|---|---|
| `TYPE` | Class, interface, enum | `@Entity class User {}` |
| `METHOD` | Methods | `@Override void fly() {}` |
| `FIELD` | Fields / member variables | `@NotNull String name;` |
| `PARAMETER` | Method parameters | `void save(@Valid User u)` |
| `CONSTRUCTOR` | Constructors | `@Inject public Service() {}` |
| `LOCAL_VARIABLE` | Local variables | `@SuppressWarnings int x;` |
| `ANNOTATION_TYPE` | Other annotations | `@Target @interface Meta {}` |
| `PACKAGE` | Package declarations | `@Deprecated package com.x;` |
| `TYPE_PARAMETER` | Type parameters (Java 8) | `class Box<@NonNull T> {}` |
| `TYPE_USE` | Any type usage (Java 8) | `@NonNull String s;` |

> If `@Target` is omitted, the annotation can be applied **anywhere** (all element types).

## 2. @Retention

Specifies **how long** the annotation is retained.

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation { }
```

### RetentionPolicy Values

```
  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
  │   SOURCE    │ ──▶ │   CLASS     │ ──▶ │  RUNTIME    │
  │             │     │             │     │             │
  │ Discarded   │     │ In .class   │     │ Available   │
  │ by compiler │     │ file but    │     │ at runtime  │
  │             │     │ NOT at      │     │ via         │
  │             │     │ runtime     │     │ Reflection  │
  └─────────────┘     └─────────────┘     └─────────────┘
```

| Policy | Scope | Used By | Example |
|---|---|---|---|
| `SOURCE` | Compiler only — gone after compilation | IDE, code generators | `@Override`, `@SuppressWarnings` |
| `CLASS` | In `.class` file — NOT available at runtime | Bytecode tools | Default if not specified |
| `RUNTIME` | Available at runtime via **Reflection** | Frameworks (Spring, Hibernate) | `@Entity`, `@Autowired` |

## 3. @Documented

Includes the annotation in **JavaDoc** output.

```java
@Documented
public @interface Important { }

@Important
public class Service { }
// "Service" JavaDoc will show: @Important annotation
```

| With `@Documented` | Without `@Documented` |
|---|---|
| Annotation appears in generated JavaDoc | Annotation hidden from JavaDoc |

## 4. @Inherited

Makes a class annotation **inheritable** — child classes automatically get the parent's annotation.

```java
@Inherited
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Important { }

@Important
class Parent { }

class Child extends Parent { }  // ← also has @Important (inherited!)
```

```java
// Verification via reflection
Child.class.isAnnotationPresent(Important.class);  // true ✅
```

| With `@Inherited` | Without `@Inherited` |
|---|---|
| Child class gets parent annotation | Child class does NOT get it |

> **Note:** Only works with **class** inheritance, NOT interfaces.

## 5. @Repeatable (Java 8)

Allows the **same annotation** to be used **multiple times** on the same element.

### Without @Repeatable (Error)

```java
@Schedule(day = "Monday")
@Schedule(day = "Friday")    // ❌ Compilation Error — duplicate
class Task { }
```

### With @Repeatable

```java
// Step 1: Create the repeatable annotation
@Repeatable(Schedules.class)             // point to container
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Schedule {
    String day();
}

// Step 2: Create the container annotation
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Schedules {
    Schedule[] value();                   // holds multiple @Schedule
}

// Step 3: Use it
@Schedule(day = "Monday")
@Schedule(day = "Friday")                // ✅ works now!
class Task { }
```

```
  @Schedule("Monday")  ─┐
  @Schedule("Friday")  ─┤──▶ internally becomes ──▶ @Schedules({@Schedule("Monday"), @Schedule("Friday")})
```

## Custom Annotations

### Creating a Custom Annotation

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {           // @interface keyword
    String name();                          // annotation member (like a method)
    int priority() default 1;              // member with default value
    String[] tags() default {};            // array member
}
```

### Using a Custom Annotation

```java
class Service {
    @MyAnnotation(name = "processOrder", priority = 5, tags = {"order", "critical"})
    public void process() { ... }
}
```

### Annotation Member Rules

| Rule | Detail |
|---|---|
| **Syntax** | Look like methods — `type name();` |
| **Allowed types** | `primitive`, `String`, `Class`, `enum`, `annotation`, arrays of these |
| **Not allowed** | Objects, collections, custom classes |
| **Default value** | `type name() default value;` — makes member optional |
| **No parameters** | Members can't have parameters |

### Reading Annotations via Reflection

```java
Method method = Service.class.getMethod("process");
MyAnnotation ann = method.getAnnotation(MyAnnotation.class);

System.out.println(ann.name());       // "processOrder"
System.out.println(ann.priority());   // 5
```

## Summary

| Annotation | Purpose |
|---|---|
| `@Target` | WHERE the annotation can be applied (METHOD, TYPE, FIELD, etc.) |
| `@Retention` | HOW LONG the annotation lives (SOURCE, CLASS, RUNTIME) |
| `@Documented` | Include in JavaDoc |
| `@Inherited` | Child class inherits parent's annotation |
| `@Repeatable` | Use same annotation multiple times (requires container) |
| **Custom** | Create with `@interface`, members with `type name()`, optional defaults |
