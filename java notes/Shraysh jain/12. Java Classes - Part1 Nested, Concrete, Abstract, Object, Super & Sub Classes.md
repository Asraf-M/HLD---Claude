## 1. Concrete Class

A class where you can **create an instance** using `new` and **all methods have implementations**.

```java
// Direct concrete class
class Person {
    void display() {
        System.out.println("I am a Person");
    }
}
Person obj = new Person();  // ✅ can instantiate

// Concrete class implementing interface
interface Shape { double area(); }

class Rectangle implements Shape {
    @Override
    public double area() { return 10 * 5; }  // ✅ implemented
}
Rectangle r = new Rectangle();  // ✅ can instantiate
```

---

## Class Access Modifiers

Unlike methods/variables, classes can only be:

| Modifier | Scope |
|---|---|
| `public` | Accessible from any package |
| *(default)* | Package-private — same package only |

> **Exception:** Nested classes can also be `private` or `protected`.

## 2. Abstract Class

Provides **abstraction** — hides implementation, exposes only features.

> **Two ways to achieve abstraction:** Interface (100%) and Abstract Class (0–100%)

```java
abstract class Car {
    abstract void pressBrake();     // abstract — no body
    abstract void pressClutch();    // abstract — no body
    
    void getNumberOfWheels() {      // concrete — has body
        System.out.println("4");
    }
}
```

### Inheritance Chain with Abstract Classes

```
  abstract Car
  ├── abstract LuxuryCar extends Car
  │     ├── provides: pressBrake() implementation
  │     └── adds: abstract pressDualBrake()
  │
  └── Audi extends LuxuryCar  (concrete)
        ├── provides: pressClutch() implementation
        └── provides: pressDualBrake() implementation
```

```java
abstract class LuxuryCar extends Car {
    abstract void pressDualBrake();           // adds more abstraction

    @Override
    void pressBrake() { System.out.println("Luxury brake"); }  // implements parent's
}

class Audi extends LuxuryCar {
    @Override void pressClutch() { ... }       // must implement
    @Override void pressDualBrake() { ... }    // must implement
}
```

### Rules

| Rule | Detail |
|---|---|
| Can have abstract + concrete methods | ✅ |
| Can be instantiated? | ❌ `new Car()` is invalid |
| Can store child reference? | ✅ `Car c = new Audi();` |
| Child must be abstract or implement all abstract methods | ✅ |

## 3. Super & Sub Class

```
  Parent class  = Superclass
  Child class   = Subclass
```

```java
class A { }          // superclass
class B extends A { } // subclass of A
```

> Every class that doesn't explicitly extend anything is implicitly a **subclass of `Object`**.

## 4. Object Class

`java.lang.Object` is the **root parent** of ALL classes in Java.

```
       Object  ← root of everything
      /      \
   Person    A
      |       |
   Doctor    B
```

### Common Object Methods

| Method | Purpose |
|---|---|
| `toString()` | String representation |
| `equals()` | Content equality |
| `hashCode()` | Hash value |
| `clone()` | Copy object |
| `wait()` / `notify()` | Thread synchronization |
| `getClass()` | Runtime class info |

```java
Object obj1 = new Person();   // ✅ Object can hold any class
Object obj2 = new Audi();     // ✅ parent of all

obj1.getClass();  // → class Person
obj2.getClass();  // → class Audi
```

## 5. Nested Classes

A class defined **inside another class**.

```
┌─────────────────── Nested Class ───────────────────┐
│                                                     │
│  ┌─── Static Nested Class ───┐                      │
│  │   (associated with class) │                      │
│  └───────────────────────────┘                      │
│                                                     │
│  ┌─── Non-Static (Inner Class) ─────────────────┐  │
│  │                                               │  │
│  │  ┌─ Member Inner ─┐                          │  │
│  │  ┌─ Local Inner ──┐  (inside a block)         │  │
│  │  ┌─ Anonymous ────┐  (no name)                │  │
│  │                                               │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**When to use:** If class A is used by **only** class B, nest A inside B instead of creating a separate file.

---

### Static Nested Class

Associated with the **outer class** (not an object). Can only access **static members** of the outer class.

```java
class OuterClass {
    int instanceVar = 10;             // instance
    static int classVar = 20;         // static

    static class NestedClass {
        void print() {
            // classVar → ✅ static
            // instanceVar → ❌ not static
            System.out.println(classVar);
        }
    }
}

// Instantiation — no outer object needed
OuterClass.NestedClass obj = new OuterClass.NestedClass();
obj.print();
```

> Nested classes can have **any access modifier** (`private`, `protected`, `public`, default) — unlike top-level classes.

---

### Inner Class (Non-Static Nested)

Also called **Member Inner Class**. Can access **all** members (static + instance) of the outer class.

```java
class OuterClass {
    int instanceVar = 10;
    static int classVar = 20;

    class InnerClass {
        void print() {
            // instanceVar → ✅
            // classVar → ✅
            System.out.println(instanceVar + classVar);
        }
    }
}

// Instantiation — NEEDS outer object first
OuterClass outer = new OuterClass();
OuterClass.InnerClass inner = outer.new InnerClass();
inner.print();
```

| | Static Nested | Inner (Non-Static) |
|---|---|---|
| Access to outer static? | ✅ | ✅ |
| Access to outer instance? | ❌ | ✅ |
| Needs outer object to create? | ❌ | ✅ |

---

### Local Inner Class

Defined inside a **method, loop, or if-block**. Only accessible within that block.

```java
class OuterClass {
    void display() {
        class LocalInner {         // local to display() method
            void print() {
                System.out.println("Inside local inner");
            }
        }
        
        LocalInner li = new LocalInner();  // ✅ only here
        li.print();
    }
}

// Cannot access LocalInner outside display()
```

| Rule | Detail |
|---|---|
| Access modifier | **Default only** (no `public`, `private`, `protected`) |
| Scope | Within the enclosing block only |
| Can access | Outer instance vars, static vars, local variables of block |

---

### Anonymous Inner Class

A class **without a name**, created inline. Used to **override behavior** without creating a separate subclass file.

```java
abstract class Car {
    abstract void pressBrake();
}

// In some method:
Car audiCar = new Car() {           // anonymous class!
    @Override
    void pressBrake() {
        System.out.println("Audi brake applied");
    }
};                                  // note the semicolon!

audiCar.pressBrake();  // "Audi brake applied"
```

**What happens behind the scenes:**

```
Compiler generates:
1. Creates a hidden class (e.g., Test$1) that extends Car
2. Implements pressBrake() with your code
3. Creates an instance of Test$1
4. Assigns reference to audiCar
```

> **Use case:** Quick, one-off implementations without cluttering codebase with small subclass files.

## Inheritance in Nested Classes

### Between Inner Classes (Same Outer)

```java
class Outer {
    class Inner1 {
        void method1() { System.out.println("Inner1"); }
    }
    
    class Inner2 extends Inner1 {    // inherits from Inner1
        void method2() {
            method1();               // ✅ inherited
        }
    }
}

Outer outer = new Outer();
Outer.Inner2 inner = outer.new Inner2();
inner.method2();
```

### Static Nested Inherited by External Class

```java
class Outer {
    static class Nested {
        void display() { System.out.println("Nested"); }
    }
}

class Other extends Outer.Nested {  // ✅ simple — static
    void test() { display(); }       // inherited
}
```

### Non-Static Inner Inherited by External Class

Requires creating outer object in constructor (because inner class is tied to an outer instance):

```java
class Outer {
    class Inner {
        void display() { System.out.println("Inner"); }
    }
}

class Other extends Outer.Inner {
    Other() {
        new Outer().super();  // must create outer first, then call super()
    }
}
```

## Summary

| Class Type | Can Instantiate? | Key Feature |
|---|---|---|
| **Concrete** | ✅ | All methods implemented |
| **Abstract** | ❌ | Can mix abstract + concrete methods |
| **Object** | — | Parent of ALL classes |
| **Static Nested** | ✅ (via `Outer.Nested`) | Access static outer members only |
| **Inner (Member)** | ✅ (via outer object) | Access ALL outer members |
| **Local Inner** | ✅ (within block only) | Scoped to method/block |
| **Anonymous** | ✅ (inline) | No name, quick override |
