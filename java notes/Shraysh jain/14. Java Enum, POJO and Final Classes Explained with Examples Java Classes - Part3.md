## 1. POJO Class

**POJO** = Plain Old Java Object — a simple class with minimal rules.

### Properties

| Rule | Detail |
|---|---|
| Contains | Variables + getter/setter methods |
| Class modifier | `public` |
| Constructor | Public default constructor |
| Annotations | **None** (`@Entity`, `@Table`, etc. not allowed) |
| Inheritance | Does NOT extend or implement anything |
| Variable access | No restrictions (can be `private`, `protected`, `public`, default) |

```java
public class Student {
    private int id;
    protected String name;
    int age;  // default

    public int getId() { return id; }
    public void setId(int id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```

### Where POJOs Are Used

**1. Request/Response Mapping** — Decouple external data from internal classes:

```
  Client Request          Component (your app)
  ┌──────────┐          ┌──────────────────────┐
  │ id: 101  │ ──────▶  │ POJO:                │
  │ name: SJ │          │   customerId: 101    │  ← mapped
  └──────────┘          │   customerName: "SJ" │
                        └──────────────────────┘
                         Used by Class1, Class2, Class3...
```

> If the request field name changes (`id` → `customer_id`), only the mapping layer changes — not all internal classes.

**2. Entity/Repository Layer** — Database table representation (though with annotations, it becomes a JPA Entity, not a pure POJO).

## 2. Enum Class

An **enum** is a special class that holds a **collection of constants** — values that are `static final` implicitly.

### Properties

| Property | Detail |
|---|---|
| Internally extends | `java.lang.Enum` (cannot extend other classes) |
| Can implement | Interfaces ✅ |
| Constructor | **Always private** (even if you write default → bytecode makes it private) |
| Cannot be instantiated | ❌ `new EnumSample()` is invalid |
| Can have | Variables, constructors, methods |
| No other class can extend it | ❌ |
| Can have abstract methods | ✅ (each constant must implement) |

### Basic Enum

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY;
}
```

Each constant gets an **ordinal** (auto-assigned starting from 0):

```
MONDAY=0, TUESDAY=1, WEDNESDAY=2, ..., SUNDAY=6
```

### Common Enum Methods

| Method | Purpose | Example |
|---|---|---|
| `values()` | Returns array of all constants | `Day.values()` |
| `ordinal()` | Returns the ordinal (index) | `Day.MONDAY.ordinal()` → `0` |
| `valueOf("name")` | Get enum by exact name string | `Day.valueOf("FRIDAY")` → `FRIDAY` |
| `name()` | Returns constant name as string | `Day.FRIDAY.name()` → `"FRIDAY"` |

```java
// Iterate over all enum constants
for (Day d : Day.values()) {
    System.out.println(d.name() + " = " + d.ordinal());
}
// MONDAY = 0, TUESDAY = 1, ..., SUNDAY = 6
```

### Enum with Custom Values

```java
public enum Day {
    MONDAY(101, "First day of the week"),
    TUESDAY(102, "Second day"),
    WEDNESDAY(103, "Mid week"),
    THURSDAY(104, "Almost there"),
    FRIDAY(105, "TGIF"),
    SATURDAY(106, "Weekend starts"),
    SUNDAY(107, "Second week off");

    private final int val;
    private final String comment;

    // Constructor (always private)
    Day(int val, String comment) {
        this.val = val;
        this.comment = comment;
    }

    public int getVal() { return val; }
    public String getComment() { return comment; }

    // Static method to find enum by custom value
    public static Day getByVal(int value) {
        for (Day d : Day.values()) {
            if (d.val == value) return d;
        }
        return null;
    }
}

Day d = Day.getByVal(107);        // → SUNDAY
System.out.println(d.getComment()); // → "Second week off"
```

### Method Override by Constant

Each constant can **override** a shared method:

```java
public enum Day {
    MONDAY {
        @Override
        public void dummyMethod() {
            System.out.println("Monday dummy");
        }
    },
    TUESDAY,
    SUNDAY;

    public void dummyMethod() {
        System.out.println("Default dummy");
    }
}

Day.FRIDAY.dummyMethod();  // → "Default dummy"
Day.MONDAY.dummyMethod();  // → "Monday dummy" (overridden)
```

### Abstract Method in Enum

If an abstract method is declared, **every constant must implement it**:

```java
public enum Day {
    MONDAY {
        @Override
        public void dummyMethod() { System.out.println("In Monday"); }
    },
    TUESDAY {
        @Override
        public void dummyMethod() { System.out.println("In Tuesday"); }
    },
    SUNDAY {
        @Override
        public void dummyMethod() { System.out.println("In Sunday"); }
    };

    public abstract void dummyMethod();
}
```

### Enum Implementing Interface

```java
interface MyInterface {
    String toLowerCase();
}

public enum Day implements MyInterface {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY;

    @Override
    public String toLowerCase() {
        return this.name().toLowerCase();  // common for all constants
    }
}

Day.MONDAY.toLowerCase();  // → "monday"
```

### Why Enum Over `static final`?

```java
// Option 1: static final constants
class WeekConstant {
    static final int MONDAY = 0;
    static final int TUESDAY = 1;
    // ...
    static final int SUNDAY = 6;
}

// Option 2: Enum
enum Day { MONDAY, TUESDAY, ..., SUNDAY; }
```

| Feature | `static final` | Enum |
|---|---|---|
| **Type safety** | ❌ Any `int` value accepted | ✅ Only defined constants |
| **Readability** | `isWeekend(6)` — what is 6? | `isWeekend(Day.SUNDAY)` — clear! |
| **Control** | Can pass invalid values (100, -1) | Only valid constants allowed |
| **Methods/Variables** | ❌ | ✅ Can have methods, constructors, fields |
| **Override per constant** | ❌ | ✅ |

```java
// static final — no control over input
boolean isWeekend(int day) { ... }
isWeekend(100);  // compiles! but meaningless

// Enum — compiler enforces valid values
boolean isWeekend(Day day) { ... }
isWeekend(100);  // ❌ Compilation Error!
```

## 3. Final Class

A `final` class **cannot be inherited** (no subclass can extend it).

```java
public final class TestClass {
    void display() { System.out.println("Final class"); }
}

class MyClass extends TestClass { }  // ❌ Compilation Error!
// Cannot inherit from final class
```

> **Use case:** When you want to **lock down** a class and prevent any modification through inheritance.

## Summary

| Class Type | Key Point |
|---|---|
| **POJO** | Simple class — vars + getters/setters, no annotations, no inheritance |
| **Enum** | Collection of constants, implicitly `static final`, always-private constructor |
| **Enum methods** | `values()`, `ordinal()`, `valueOf()`, `name()` |
| **Enum custom values** | Add fields + parameterized constructor |
| **Enum advantage** | Type safety, readability, control over `static final` |
| **Final Class** | Cannot be inherited — `final class X { }` |
