## Future Interface

Represents the **result of an async task**. Allows the caller to check status, get result, or cancel.

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(...);

Future<?> future = executor.submit(() -> {
    // async task — runs in separate thread
    Thread.sleep(7000);
});

// main thread continues...
future.isDone();     // false — still running
future.get();        // BLOCKS until task completes
future.isDone();     // true
```

```
  Main Thread              Thread-1 (async)
       │                        │
       ├── submit(task) ──────▶ │ (starts working)
       │   returns Future       │
       │                        │
       ├── isDone()? → false    │ (still working...)
       │                        │
       ├── get() ──── BLOCKS ──▶│ (still working...)
       │             WAITING    │ (done!)
       │◄───────── result ──────┘
       │
       ├── isDone()? → true
```

## Future Methods

| Method | Return | Description |
|---|---|---|
| `cancel(boolean)` | `boolean` | Attempt to cancel. Returns false if already completed |
| `isCancelled()` | `boolean` | Was the task cancelled? |
| `isDone()` | `boolean` | Is the task done? (completed, cancelled, or exception) |
| `get()` | `T` | **Blocks** until complete, then returns result |
| `get(timeout, unit)` | `T` | Blocks for max `timeout` — throws `TimeoutException` if not done |

```java
// Timed get — don't block forever
try {
    Object result = future.get(2, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    System.out.println("Task not done in 2 seconds");
}

// Then wait indefinitely
Object result = future.get();  // blocks until done
```

## Runnable vs Callable

| Feature | Runnable | Callable |
|---|---|---|
| Method | `void run()` | `V call() throws Exception` |
| Return value | ❌ No return | ✅ Returns a value |
| Exceptions | Cannot throw checked | Can throw checked exceptions |
| Future result | `Future<?>` (always null) | `Future<V>` (actual result) |

```java
// Runnable — no return
executor.submit(() -> {
    System.out.println("Working...");
});

// Callable — returns value
Future<Integer> future = executor.submit(() -> {
    return 42;  // ← return makes it Callable
});
Integer result = future.get();  // 42
```

## Three Flavors of submit()

### 1. submit(Runnable) — No Return Value

```java
Future<?> future = executor.submit(() -> doWork());
future.get();  // null (Runnable has no return)
```

### 2. submit(Runnable, T result) — Shared Object Workaround

```java
List<Integer> output = new ArrayList<>();

Future<List<Integer>> future = executor.submit(() -> {
    output.add(300);   // modifies shared object
}, output);            // ← returns this object when done

List<Integer> result = future.get();  // [300]
```

### 3. submit(Callable) — Clean Return Value

```java
Future<String> future = executor.submit(() -> {
    return "Hello World";  // ← Callable because it returns
});

String result = future.get();  // "Hello World"
```

## CompletableFuture

An **advanced version of Future** (Java 8) that supports **chaining** of async operations.

```
  Future           → check status, get result, cancel
  CompletableFuture → all of Future + chaining + compose + combine
```

```java
// CompletableFuture implements Future
public class CompletableFuture<T> implements Future<T>, CompletionStage<T> { }
```

### Default Thread Pool

| Method | Thread Pool Used |
|---|---|
| No executor passed | `ForkJoinPool.commonPool()` (default) |
| Executor passed | Your custom `ThreadPoolExecutor` |

## CompletableFuture Methods

### 1. supplyAsync — Initiate Async Operation

```java
CompletableFuture<String> future = CompletableFuture.supplyAsync(
    () -> {
        // async task — returns result
        return "Hello";
    },
    executor  // optional — uses ForkJoinPool if omitted
);

String result = future.get();  // "Hello"
```

### 2. thenApply / thenApplyAsync — Transform Result

Applies a **function** to the result of the previous stage.

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Concept")                    // Thread-1: returns "Concept"
    .thenApply(val -> val + " and Coding");           // SAME Thread-1: returns "Concept and Coding"

future.get();  // "Concept and Coding"
```

| Method | Thread Used |
|---|---|
| `thenApply` | **Same thread** as previous stage (synchronous) |
| `thenApplyAsync` | **New thread** from pool (asynchronous) |

```java
// thenApplyAsync — different thread
.thenApplyAsync(val -> val + " and Coding", executor);
```

### 3. thenCompose / thenComposeAsync — Chain Dependent Async Ops

Chains **dependent** async operations with **guaranteed ordering**.

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")
    .thenCompose(val -> CompletableFuture.supplyAsync(
        () -> val + " World"
    ))
    .thenCompose(val -> CompletableFuture.supplyAsync(
        () -> val + "!"
    ));

future.get();  // "Hello World!" — always in order
```

| Method | Thread | Ordering |
|---|---|---|
| `thenCompose` | Same thread | ✅ Guaranteed |
| `thenComposeAsync` | New thread | ✅ Guaranteed (internal dependency stack) |

### 4. thenAccept / thenAcceptAsync — End of Chain (No Return)

Consumes the result **without returning anything** — typically the last step.

```java
CompletableFuture.supplyAsync(() -> "Result")
    .thenApply(val -> val.toUpperCase())
    .thenAccept(val -> System.out.println(val));  // prints "RESULT"
    // returns CompletableFuture<Void> — can't chain thenApply after this
```

### 5. thenCombine / thenCombineAsync — Combine Two Futures

Combines results from **two independent** CompletableFutures.

```java
CompletableFuture<Integer> task1 = CompletableFuture.supplyAsync(() -> 10);
CompletableFuture<String>  task2 = CompletableFuture.supplyAsync(() -> "K");

CompletableFuture<String> combined = task1.thenCombine(
    task2,
    (num, str) -> num + str    // BiFunction<Integer, String, String>
);

combined.get();  // "10K"
```

```
  task1 (Thread-1) → 10 ─┐
                           ├── thenCombine → "10K"
  task2 (Thread-2) → "K" ─┘
```

### Quick Reference

| Method | Input | Output | Thread | Use Case |
|---|---|---|---|---|
| `supplyAsync` | `Supplier<T>` | `CompletableFuture<T>` | New | Start async task |
| `thenApply` | `Function<T,R>` | `CompletableFuture<R>` | Same | Transform result |
| `thenApplyAsync` | `Function<T,R>` | `CompletableFuture<R>` | New | Transform in new thread |
| `thenCompose` | `Function<T, CF<R>>` | `CompletableFuture<R>` | Same | Chain dependent async |
| `thenAccept` | `Consumer<T>` | `CompletableFuture<Void>` | Same | End of chain |
| `thenCombine` | `CF<U>, BiFunction` | `CompletableFuture<R>` | Same | Merge two futures |

## Summary

| Concept | Key Point |
|---|---|
| **Future** | Track async task status — `isDone()`, `get()`, `cancel()` |
| **Runnable** | No return value — `Future<?>` always null |
| **Callable** | Returns value — `Future<T>` with actual result |
| **CompletableFuture** | Advanced Future — supports chaining |
| **supplyAsync** | Start async task (like `submit()`) |
| **thenApply** | Transform result — same thread |
| **thenApplyAsync** | Transform result — new thread |
| **thenCompose** | Chain dependent async ops — ordering guaranteed |
| **thenAccept** | Consume result — end of chain, no return |
| **thenCombine** | Merge results from two independent futures |
